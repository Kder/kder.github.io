提高Firefox3地址栏反应速度 八月 31st, 2009

Firefox3最好用的功能，莫过于它的智能地址栏了。但用的时间久了，就会觉得反应会变慢，今天分享一个小技巧：如何提高Firefox3地址栏的反应速度。

基本思路：利用javascript代码整理sqlite数据库

方法 ：在工具—错误控制台—消息 的代码框中输入如下代码，回车

Components.classes["@mozilla.org/browser/nav-history-service;1"].getService(Components.interfaces.nsPIPlacesDatabase).DBConnection.executeSimpleSQL("VACUUM");

解决Mplayer编译时出现的ivtv错误 六月 27th, 2009

在Slackware下编译安装Mplayer时出现了如下错误：

vo_ivtv.c: In function 'ivtv_reset': vo_ivtv.c:79: error: storage size of 'sd' isn't known vo_ivtv.c:80: error: storage size of 'sd1' isn't known vo_ivtv.c:84: error: 'IVTV_STOP_FL_HIDE_FRAME' undeclared (first use in this function) vo_ivtv.c:84: error: (Each undeclared identifier is reported only once vo_ivtv.c:84: error: for each function it appears in.) vo_ivtv.c:87: error: 'IVTV_IOC_STOP_DECODE' undeclared (first use in this function) vo_ivtv.c:97: error: 'IVTV_IOC_START_DECODE' undeclared (first use in this function) vo_ivtv.c:80: warning: unused variable 'sd1' vo_ivtv.c:79: warning: unused variable 'sd' make[1]: *** [vo_ivtv.o] Error 1 make[1]: Leaving directory `/home/kder/MPlayer-1.0rc2/libvo' make: *** [libvo/libvo.a] 错误 2

解决方案：在./configure 时后面加上 --disable-ivtv 即可，下面是我的配置：

./configure --enable-gui --codecsdir=/usr/lib/codecs --win32codecsdir=/usr/lib/codecs --realcodecsdir=/usr/lib/codecs --xanimcodecsdir=/usr/lib/codecs --language=zh_CN --disable-ivtv

转载请注明 作者 Kder 出处 http://www.kder.info/mplayer-ivtv-error.html

几个Slackware小问题的解决 六月 25th, 2009

［本文实践环境为 Slackware 12.2 + Xfce］

问题：如何让程序随xfce自动启动

解决方案：终端启动 xfce4-autostart-editor  ， scim输入法也可以在这里设置自启动

问题：如何设置Locale

解决方案：在~/.profile中设置，例如 export LANG=zh_CN.gbk

问题：设置为中文locale之后，启动mousepad之类的程序特别慢，开始还以为是locale的原因，后来查明是每次启动gtk程序的时候都会调用scim输入法，而退出程序后输入法也退出了，所以每次启动都很慢

解决方案：用xfce4-autostart-editor让scim -d 随xfce系统启动

问题： 每次手动设置～/.xinitrc后，例如export 变量、设置输入法启动等等，startx都是一闪而过（x默认的窗口管理器是xfce，用xwmconfig设置的），后查明，原来每次启动x都要调用 /etc/X11/xinit/xinitrc.xfce这个文件，其中包含了启动xfce的过程，如启动session、panel等等，手动设 置.xinitrc后，因.xinitrc优先级比/etc/X11/xinit/xinitrc.xfce高，后者被override了，所以不再执行 xinitrc.xfce中的内容，而执行.xinitrc，如果其中只有export或者输入法之类的程序，当然屏幕就一闪而过了。

解决方案：启动程序在xfce4-autostart-editor 中设置，export变量在~/.profile中设置

问题：自带的Terminal程序不能使用（不能输入任何字符）

解决方案：root权限执行： chsh -s /usr/bin/bash <username> 参考：http://www.linuxsir.com/bbs/thread347458.html

问题：Terminal没有彩色显示文件名

解决方案：~/.bashrc里加入： alias ls='ls -F --color=auto' alias ll='ls -l -h --color=auto' alias la='ls -A --color=auto'

参考：http://www.linuxsir.org/bbs/thread263743.htm

http://www.macfans.com.cn/viewthread.php?tid=34827&highlight=Terminal%2B%B2%CA%C9%AB

转载请注明    作者：Kder 出处：http://www.kder.info/slackware-little-problems.html
